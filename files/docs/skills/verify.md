# 스킬: 기능 검증 (Verify)

검증 워크플로우의 표준 진입점. 구현과 반드시 별도 세션에서 수행한다.

## 사전 조건 (사람이 확인)
- [ ] 구현 에이전트의 완료 보고가 있는가 (사람이 확인 — verifier는 읽지 않음)
- [ ] feature 파일의 "구현 위치" 섹션이 작성되어 있는가
- [ ] docs/index.md 상태가 `검증중`인가
- [ ] 이 세션은 구현 세션과 다른 새 세션인가 (필수)

## 시작 방법
```
새 Claude Code 세션을 열고 다음을 입력:
"verifier 역할로 [기능명] 검증해줘"
```

## (참고) 에이전트 수행 순서
에이전트는 역할 파일(verifier.md)의 지시에 따라 아래 순서로 수행한다.
이 섹션은 사람이 흐름을 파악하기 위한 참고용이며, 에이전트가 이 파일을 읽지 않아도 된다.

1. CLAUDE.md 읽기
2. docs/roles/verifier.md 읽기
3. docs/features/[기능명].md 읽기 (완성 기준 + 구현 위치)
4. 구현된 코드 읽기 (구현 위치 기준)
5. docs/shared.md 읽기 (의존성 명시된 경우)
6. 시도 번호 결정 (failure-report 파일 수 기준)
7. 각 완성 기준 판정
8. 결과 보고 및 failure-report 처리

## 완료 후
검증 결과에 따른 상태 전이·다음 단계·시도 카운터 리셋 규칙은 docs/workflow.md 의 "상태 전이"/"전체 흐름"이 SSOT.
- 요약: PASS → 완료 / FAIL(1~2회) → 구현중(새 세션 재시도) / FAIL 3회 → 에스컬레이션
