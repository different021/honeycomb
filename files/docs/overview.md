# 프로젝트 맥락 (Overview)

> 새 프로젝트 시작 시 사용자에게 물어 채운다. 비어 있으면(`(미작성)`) 메인 세션이 본격 작업 전 채운다.
> 하위 역할 세션(implementer/verifier)은 이 파일 전체를 읽지 않는다 — 필요한 섹션만 (각 role 파일 "읽어야 할 것" 참고).

## 프로젝트 개요
> 업무/도메인 맥락. 무엇을, 왜 만드는가. 단위 작업 에이전트는 보통 읽지 않아도 된다(컨텍스트 절약).

(미작성 — 프로젝트 시작 시 사용자에게 물어 채운다)

## 기술 컨텍스트
> 스택 · 빌드/테스트 명령 · 코드 배치 규칙. implementer/verifier가 구현·검증에 필요할 때 읽는다.
